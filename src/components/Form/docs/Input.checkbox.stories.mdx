# Checkbox

Checkbox should be used for item/option selection a,donly when more than one item can be selected at the same time.
If there is only two options (on/off) consider using a switch.
 
If the user should select only one option, use radio buttons.

Use a checkbox when you want to give your recipients the opportunity to select more than one option.

## Content
Checkboxes appear as a list on UI so it's important to make their labels as parallel as possible without twisting the language too much. It’s important to have parallelism, but not at the expense of comprehension. Labels should appear to the right of checkboxs.

<Use>
	<Use.Do>
		<ul>
			<li>Use positive and active wording for checkbox lables.</li>
			<li>Aim for parallel checkbox lables to help users make choices and to ease reading flow.</li>
			<li>Start each label with capital letter.</li>
		</ul>
	</Use.Do>
	<Use.Dont>
		<ul>
			<li>Don’t use punctuations at the end of labels.</li>
		</ul>
	</Use.Dont>
</Use>

## Interaction

- If a label is provided, it should always be clickable to check or uncheck.
- Checking one checkbox does not change the state of others, unless the parent was indeterminant and all children have the same state.

## Accessibility

Press <kbd>TABS</kbd> to focus on a checkbox.

Press <kbd>TABS</kbd> or <kbd>Shift TABS</kbd> to navigate between checkboxes.

Press the <kbd>ENTER</kbd> or <kbd>SPACE</kbd> to toggle the checkbox between selected and not selected.
