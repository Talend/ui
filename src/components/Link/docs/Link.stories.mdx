# Link

Link is used for navigation.

If you use a link for triggering an action on the page, you must consider using a [Button](/?path=/docs/components-button--primary) instead.

Still need to be convinced? Please, take the time to read [this article written by Marcy Sutton](https://marcysutton.com/links-vs-buttons-in-modern-web-applications).

When the link targets a new domain, we add an external-link icon right after.

The attribute `rel="noopener noreferrer"` is added if its target is `_blank`.

## Content

Link labels should be specific and clear enough to increase the likelihood that users will quickly understand the link as they scan and process the page. There's no maximum word count for links, but the best practice is to be as succinct as possible.

<Use>
	<Use.Do>
		<ul>
			<li>
				Use a meaningful link text which communicates to users what they'll exactly find on the
				other side of the click.
			</li>
			<li>
				Make link text stand aloneâ€”it should make sense without any of the surrounding text as users
				tend to scan UI.
			</li>
			<li>
				Frontload link text with keywords and use the rest of the label text to increase user
				understanding.
			</li>
		</ul>
	</Use.Do>
	<Use.Dont>
		<ul>
			<li>
				Avoid vague or repetative text for links. For example, don't use the text "Click here" or
				"See this document" for links.
			</li>
			<li>Don't use a URL as link text.</li>
		</ul>
	</Use.Dont>
</Use>

## Interaction

Text underline appears while hovering the link and disappears as soon as the mouse is out of its clickable area with a smooth animation.

## Accessibility

Press <kbd>TABS</kbd> to focus on a link

Press <kbd>ENTER</kbd> to open a link
